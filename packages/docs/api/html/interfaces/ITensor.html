    Interface ITensor

    Represents a handle to a Tensor computation.

    This interface follows the "Recursive Computer" philosophy. An ITensor instance is not the data itself, but a handle to a computation unit that resides in some backend (CPU, WebGPU, Wasm, etc).

    All mathematical operations return a new ITensor handle synchronously (building the graph/queue), while raw data access is asynchronous.
