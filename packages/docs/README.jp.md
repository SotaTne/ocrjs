# Docs

## 概要

`docs` は **ocrjs に関する公式ドキュメントを提供するためのパッケージ**です。

このパッケージは単なる Web 用ドキュメントではなく、  
**ocrjs の設計思想・利用方法・契約を一貫した形で配布・参照可能にすること**を目的としています。

---

## パッケージとして提供する理由

`docs` を独立したパッケージとして扱う理由は、  
ドキュメントを **単なる閲覧物ではなく、再利用可能な成果物**として位置付けているためです。

主な用途は以下の通りです。

### 1. ドキュメントサイトとしてのホスティング

- Web 上で公式ドキュメントとして公開する
- ocrjs の全体像・設計思想・利用方法を一元的に提供する

---

### 2. ローカル環境での利用

- パッケージをインストールするだけでドキュメントを閲覧可能
- オフライン環境や制限された環境でも参照できる
- 学習・検証用途での利用を想定

---

### 3. MCP（Machine-Consumable Package）としての利用

- ドキュメント構造を機械的に解釈可能な形で提供
- LLM やツールチェーンからの参照・補助利用を想定
- 将来的な自動生成・検索・ナビゲーションとの親和性を確保

---

## Docs の役割

`docs` は次の役割を担います。

- ocrjs の **公式な知識の唯一の正解（Single Source of Truth）**
- 設計・概念・API の背景を含めた文脈の共有
- 実装を超えて **思想を伝えるための層**

そのため、`docs` はコードの補足ではなく、  
**ocrjs というプロダクトそのものの一部**として扱われます。

---

## 補足

`docs` の内容自体は一般的なドキュメントと同様に構成されますが、  
配布形態・利用方法・将来の拡張性を考慮し、  
パッケージとして管理されている点が特徴です。

---

## まとめ

- `docs` は ocrjs の公式ドキュメントパッケージ
- Web・ローカル・機械利用のすべてを想定している
- ドキュメントを「成果物」として扱う設計
- ocrjs の思想と契約を伝えるための重要な構成要素
