# ocr

## 概要

`ocr` は **ocrjs における標準的な OCR 実行パッケージ**です。

このパッケージは、`core` が定義する抽象的な OCR ドメインに対して、  
**実用上必要となる依存関係をあらかじめ解決した、最小かつ標準的な実装**を提供します。

言い換えると、  
**特別な構成やカスタマイズを行わずに OCR を実行したい場合に使用するパッケージ**です。

---

## 役割と位置づけ

`ocr` は次の特徴を持ちます。

- `core` の **完全なラッパー**
- OCR 実行に必要な構成を **あらかじめ束ねた参照実装**
- 「まず使うならこれ」という **デフォルト選択肢**
- 柔軟性を意図的に残している **最小限の完成形**

`ocr` 自体は新しいドメインを定義せず、  
`core` が定義する契約に **忠実に従う実装のみ**を提供します。

---

## 提供するもの

`ocr` パッケージが提供するのは、**OCR の実行そのもの**です。

- OCR インスタンスの生成
- 入力データを受け取り、認識結果を返す
- `core` が定義するライフサイクルと不変条件の遵守

これ以上でも、これ以下でもありません。

---

## 提供しないもの（重要）

`ocr` は、意図的に以下の機能を **提供しません**。

- 画像の取得（`fetch` / ファイル I/O）
- 画像前処理・描画・可視化
- 認識結果のオーバーレイ表示
- UI やフレームワーク向けの補助機能
- 特定の実行環境（Web / Node / Bun など）への最適化

これらは、ocrjsの提供する他のパッケージ・OCR実行可能なパッケージが担うべき責務です。

---

## 設計意図

`ocr` は、次の設計方針に基づいています。

- OCR という能力を **純粋な形で提供する**
- 利用者に余計な選択や判断を強制しない
- 上位レイヤが自由に拡張できる余地を残す
- 実装詳細を隠蔽し、契約のみを公開する

このため、`ocr` は「便利機能の集合」ではなく、  
**ocrjs における最小の完成形**として設計されています。

---

## まとめ

- `ocr` は ocrjs における **標準的な OCR 実行パッケージ**
- `core` の完全ラッパーとして機能する
- OCR の実行以外の責務を持たない
